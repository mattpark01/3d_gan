The Use of Adversarial Criterion to Generate 3D objects

Project Proposal by Matthew Park

Abstract

The core idea for my project is to be able to generate three-dimensional object shapes through a generative deep learning model without the involvement of supervision during training.

The importance of the adoption for efficient three-dimensional object generation is critical particularly in the event of when the internet's interface transitions to become multi-dimensional (or immersed into the life of humanity, e.g., mixed-reality). However, putting the future of the internet's interface aside, this is critically important even today. An example of why three-dimensional object generation is important today is for game developers to generate game assets and objects in an efficient manner and more relevant for the average business: efficiently generating three-dimensional representations of products for eCommerce stores.

Today there is no current implementation of being able to generate three-dimensional objects in photorealistic quality, let alone with inclusion of dynamic functions, events, and interactions. This project serves as the foundational techniques that will be required for the scalability of the three-dimensional object generation.

Project Background

Inspiration of my approach and data for my implementation is credited to the paper, "Learning a Probabilistic Latent Space of Object Shapes via 3D Generative-Adversarial Modeling", by Jiajun Wu, Chengkai Zhang, Tianfan Xue, William T. Freeman, and Joshua B. Tenenbaum.

[1] They approached the problem of 3D object generation. Their framework, namely 3D Generative Adversarial Network (or 3D-GAN) can generate 3D objects from a probabilistic space. As mention in their abstract for their paper, the benefits of their model are three-fold:

  1. By using an adversarial criterion it allows the generator to create high-quality 3D objects by capturing the object structure implicitly, rather than relying on traditional heuristic criteria.
  1. The generator creates a way to sample and explore 3D objects without relying on reference images or CAD models by mapping a low-dimensional probability space to the 3D object space.
  1. Their approach for generating high-quality 3D objects achieve impressive performance in 3D object recognition by using an adversarial discriminator for unsupervised learning which results in the comparability with those of supervised learning methods.

High Level Technical Description for Project

An adversarial discriminator is a key component in Generative Adversarial Networks (GAN)

The purpose of the generator is to generate, based off a given set of data, the most realistic possible 3D object samples. The purpose of the discriminator is to determine whether a given input is real or fake. In a generative adversarial network (GAN), there are two neural networks that are being trained together: the generator and the discriminator.

The generator is responsible for generating new samples that are similar to a provided dataset, while the discriminator tries to distinguish between real and fake samples. The two networks are trained in a way that the generator is constantly trying to create samples that are realistic enough to fool the discriminator, while the discriminator is trying to become more accurate in distinguishing between real and fake samples.

The adversarial nature of the discriminator comes from the fact that it is trained to be highly discriminative, which means that it is constantly trying to improve its ability to distinguish real from fake samples, in order to make it more difficult for the generator to create samples that can fool it. In other words, the discriminator is an adversary of the generator, and its purpose is to push the generator to create more realistic samples. This adversarial training process is what makes GANs so effective at generating highly realistic and novel samples.

Model Evaluation and Implementation

In regards, to training, it is good protocol to update both generator and the discriminator in every batch. If we take a step back we can intuitively determine that generating objects in 3D voxel space by the generator is more computationally expensive than differentiating between real and synthetic objects by the discriminator. In simple terms, this means the discriminator is prone to being way ahead of the generator so it is essential to implement a training strategy that keeps the two networks on pace with each other.
